# i18n / Localización (app móvil)

La app usa **i18next + react-i18next**. La **fuente de verdad es el español**
(`locales/es/translation.json`); el resto de idiomas son traducciones de ese.

## Arquitectura

- `locales/<lang>/translation.json` — cadenas por idioma (claves namespaced).
- `lib/i18n/index.ts` — init de i18next (`fallbackLng: "es"`, detección por Intl).
- `lib/i18n/languages.ts` — `LANGUAGES` (registro) + `detectDeviceLang()` + `localeForLang()`.
- `lib/i18n/language-context.tsx` — `LanguageProvider` + `useLanguage()` (pref + idioma
  efectivo, persistido en SecureStore `nidokey.language`).
- `components/LanguageSelector.tsx` — selector en Ajustes (Cuenta → Idioma).
- `types/i18next.d.ts` — claves **tipadas** desde el `es` (typos = error de compilación).
- Formateo (moneda/fecha/número): `packages/shared/src/format.ts` con
  `setFormattingLocale()`, que `LanguageProvider` fija al cambiar de idioma.

`fallbackLng: "es"`: si una clave falta en otro idioma, se ve en español (nunca la
clave cruda). El validador avisa de las que falten.

## Cómo USAR una cadena en un componente

```tsx
import { useTranslation } from "react-i18next";
const { t } = useTranslation();
<Text>{t("common.save")}</Text>
<Text>{t("records.empty", { type })}</Text>      {/* interpolación */}
<Text>{t("matches.dropCount", { count })}</Text>  {/* plural _one/_other */}
```

Reglas:
- **No concatenar** cadenas: usar interpolación `{{var}}`.
- Plurales: `clave_one` / `clave_other` (i18next, Intl.PluralRules).
- Claves descriptivas y estables: `common.save`, `matches.merge_button`.

## Cómo AÑADIR una cadena nueva

1. Añádela primero a `locales/es/translation.json` (fuente).
2. Añade la traducción en `locales/en/translation.json` (y demás idiomas).
3. Úsala con `t("ns.key")` — `tsc` exige que exista en `es`.
4. `npm run i18n:validate` para confirmar paridad de claves.

## Cómo AÑADIR un idioma

1. Crea `locales/<code>/translation.json` (copia de `es` traducida, o
   `npm run i18n:translate -- --lang <code> --api google` para un borrador).
2. Regístralo en `lib/i18n/index.ts` (`resources`) y en `lib/i18n/languages.ts`
   (`LANGUAGES` + `SUPPORTED`).
3. `npm run i18n:validate`.

## Scripts

- `npm run i18n:validate` — comprueba que cada idioma tiene todas las claves de `es`
  (exit ≠0 si falta algo; apto para pre-build/CI).
- `npm run i18n:translate -- --lang en,pt-BR --api deepl` — borradores automáticos de
  las claves que falten (requiere `DEEPL_API_KEY` o `GOOGLE_TRANSLATE_API_KEY`).
  Son **borradores**: marca el idioma como `translationQuality: "automatic"` y revisa.

## Pendiente / fuera de alcance actual

- Idiomas activos: **es, en** (la infra admite más con 1 JSON).
- **RTL** (árabe/hebreo): el flag `isRTL` existe en `LANGUAGES`, pero el trabajo de
  layout RTL está aplazado (sin idiomas RTL activos).
- Web (`src/`): es la landing de descarga; no se internacionaliza aquí.
- Extracción de cadenas: en curso por tandas (pantallas de más tráfico primero); las
  no extraídas se ven en español por el fallback hasta migrarlas.
